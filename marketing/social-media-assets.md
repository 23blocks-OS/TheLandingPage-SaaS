## 11. Screenshot Guidelines

### Taking Screenshots

**General Guidelines**:
- **Resolution**: 2x (Retina) for clarity
- **Browser**: Chrome (clean, no extensions)
- **Zoom**: 100% or 125%
- **Window**: Full screen or consistent size
- **Background**: Clean desktop

**Dashboard Screenshots**:
- Show populated data (not empty states)
- Use realistic project names
- Show success states
- Hide sensitive info (emails, tokens)

**Terminal Screenshots**:
- Use clean theme (Dracula, Nord, or custom)
- Larger font size (16-18pt)
- Show full commands and output
- Include success indicators

**Tools**:
- **macOS**: Cmd+Shift+4 (built-in)
- **Windows**: Snipping Tool / Snip & Sketch
- **Linux**: Flameshot, Spectacle
- **Browser**: GoFullPage (extension)

### Editing Screenshots

**Add annotations**:
- Arrows pointing to features
- Numbered steps
- Highlight important text
- Add callout boxes
- Blur sensitive info

**Tools**:
- Skitch (macOS)
- Greenshot (Windows)
- Pinta (Linux)
- Figma (all platforms)
